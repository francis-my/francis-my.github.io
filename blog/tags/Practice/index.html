如何使用github免费搭建自己的博客网站

April 2, 2025 · 5 min read
Koala
Koala is a cybersecurity engineer

1.安装git​

2.安装node.js​

3.选个建站框架-本地部署​

本人这里打算做个博客网站，看朋友搭建的Docusaurus界面样式都挺好看的。也就选择了它。

安装 | Docusaurus中文文档 | Docusaurus中文网

#在线安装
npx create-docusaurus@latest my-blog classic --javascript

#安装完成后，可以直接在本地先部署
cd my-blog
npm run start

#生成静态网站
npm run build

#生成的静态网站的文件都在 my-blog/build 目录下
#后续只要把build下的文件上传到github上就可以。上传至git暂存区的 也是这部分。

4.git配置SSH key​

相信看到本文的你应该听过github，并且也注册了自己的github账号，如果现在还没有的可以去github注册一个账号。

为什么要配置SSH key？ 目的：可以免密的将本地的源码和资源上传到github，无需要每次都输账号和密码。

先看本地是否配置好SSH key（命令：cd ~/.ssh）

这里我之前就配置过了，配置过程如下：

#生成ssh key

ssh-keygen -t rsa -C "邮件地址"

#备注：这里的邮件地址是github账号绑定的邮件地址
#输入生成命令：ssh-keygen -t rsa -C "邮件地址"后。连续回车三次即可

可以看到在用户文件夹下生成了一个.ssh文件夹

进入到ssh文件夹，复制id_rsa.pub文件里全部内容

接着打开github主页，点击个人设置，点击左侧的SSH and GPG keys，点击New SSH key

5.部署到github上​

新建一个github仓库

给仓库起个名称，注意！注意！注意！

仓库名称格式： 用户名.github.io

因为这个仓库名称将作为你github博客的访问地址

回到本地项目文件，右键Git Bash Here（以下操作属于Git的知识，先上传至暂存区/本地分支，再上传至Git仓库分支中/Github上）

#使用命令git init 初始化生成.git
git init

#使用命令git add . 将所有文件提交到暂存区（add后面跟一个点，意思是当前目录下的所有文件）
git add .

#使用命令git commit -m 'first commit' 备注
git commit -m 'first commit'

#使用命令git remote add origin git@github.com:用户名/用户名.github.io.git 
git remote add origin git@github.com:用户名/用户名.github.io.git

#使用git push -u origin main 将项目提交到Github上
git push -u origin main

6.可能遇到的问题​

1.本地分支和远程分支命名不一致

 #查看本地分支信息（名称）
 git branch
 
 #将本地分支master重命名为main
 git branch -m master main

为何重命名为 main 后推送成功？ GitHub 远程分支默认分支名可能是 main，但远程没有 master 分支。当您推送 master 时，GitHub 自动新建了 master 分支，因此没有冲突。

2.Error: failed to push some refs to xxx

 #完全覆盖远程main分支（远程仓库中的所有文件都会被覆盖掉）
 git push -u origin main --force

7.首次推送完成后，再次推送的操作如下：​

#重新生成静态网站
npm run build
#将build下的所有文件提交至暂存区
git add .
#备注第几次提交
git commit -m 'second commit'
#提交到github上
git push -u origin main

8.注意​

在这里，作者建议 上传暂存区 的文件，单独存放在一个文件夹内。例如：

'My-Blog' 是本地部署的项目文件

'francis-my.github.io-main' 是存放静态网站的内容文件（该目录下的文件来源于My-Blog/build。每次生成新的静态网站时，都需要覆盖该文件夹）
